Roller Coaster Function

Cubic Function

Transformations

The parent cubic function was transformed to model the motion of a roller coaster in Disney Park. The graph opens downward, symbolizing the initial drop after the peak.

It was vertically compressed by a factor of 0.158, horizontally stretched by 1.672, and translated 0.89 units left and 0.47 units upward to match the coaster’s base and slope.

Domain and Range

The domain is approximately −6 < x < −1.7, representing the section of track from the start of the climb to the end of the descent.

The range corresponds to the coaster’s height in this portion, emphasizing a realistic part of the ride.

Support Frame of the Pirate Ship Ride

Absolute Function

Transformations

The parent absolute value function was transformed to model the “V”-shaped support frame of the pirate ship ride. It is reflected vertically, stretched, and shifted to match the frame’s position.

Domain and Range

Domain: −4 < x < 5, modeling the horizontal span of the beams.

Range: −3.2 ≤ y ≤ 3.3, reflecting the vertical height of the support frame realistically.

Top of Roller Coaster Track

Sinusoidal Function

Transformations

The sine function was vertically stretched by 1.5, horizontally stretched by 2.22, shifted 2.8 units left, and moved downward by 0.5 units to match the top section of the coaster track.

Domain and Range

Domain: -5 < x < 5, focusing on the crest of the track.

Range: -5 ≤ y ≤ 1, representing the physical limits of the coaster’s top arc.

Downhill Section of Roller Coaster Track

Exponential Function

Transformations

The exponential function f(x) = 2^x was reflected vertically, compressed by 0.2, horizontally stretched by 3.33, shifted 18.1 units left, and translated upward by 6.9 units to match the downhill track.

Domain and Range

Domain: -2 < x < 0.8, Range: -3.24 ≤ y ≤ 1.24, representing the physical extent of this downhill section.
